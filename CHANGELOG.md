# Changelog

本文件记录 GWShell 的版本变更。格式参考 [Keep a Changelog](https://keepachangelog.com/)，
版本号遵循 [语义化版本](https://semver.org/)。

发布说明由 CI（`.github/workflows/build.yml`）按标签版本号自动从本文件提取，
因此每个版本的小节标题必须形如 `## vX.Y.Z - YYYY-MM-DD`。

## v0.5.1 - 2026-06-13

### ✨ 新功能
- **按连接系统自动选择补全命令表**：命令补全下拉不再一律使用 Linux 命令列表，而是依据所连接的系统选择对应命令表。本地终端按 Shell 类型识别（CMD / PowerShell / Bash 等）；SSH 连接在登录后通过 `uname` / `%COMSPEC%` 探测远端系统，也可在资产「高级」设置中手动指定「远程 Shell（自动补全）」。新增 Windows CMD 与 PowerShell 两套命令表（含中英文说明），未识别时回退到原有的 Unix/Linux 表。

## v0.5.0 - 2026-06-11

新增多项实用功能，补完此前的半成品入口，清理冗余代码与依赖，并对资产列表做了系统的界面打磨。

### ✨ 新功能
- **SSH config 导入**：从 `~/.ssh/config` 一键导入主机为 SSH 资产（解析 Host / HostName / Port / User / IdentityFile / ProxyJump，跳过通配符与 Match 块），同名资产自动跳过。设置 → 储存仓库，以及空状态引导卡均有入口。
- **SFTP 文件夹上传 / 下载 + 传输进度**：支持递归传输整个目录；所有上传 / 下载（含单文件）实时显示进度条（当前文件、序号、百分比）。
- **会话日志**：设置开关，开启后终端输出（去除控制字符）按天追加保存到应用数据目录的 `logs/{会话名}-{日期}.log`。
- **标签拖拽排序**：终端标签可拖拽重排（首页标签固定）。
- **储存仓库接线**：导出 / 导入 / 清除本地数据三个按钮真正可用（导出文件不含密码等敏感信息，清除带二次确认）。
- **Docker 测试连接**：新建 Docker 主机时可一键探测容器列表，验证连接是否可用。

### 🎨 界面打磨（资产列表）
- 未连接状态由刺眼的红点改为中性灰点（红色保留给真实故障）。
- 每张资产卡片都显示**类型徽章**（SSH / Docker / 串口 / 本地终端），SSH 卡额外并排显示延迟；docker / 串口 / 本地终端不再显示无意义的「超时」。
- 卡片紧凑化、列表视图按分组分节并自动隐藏全空列、零资产时显示三个引导 CTA。
- 延迟配色阈值调整为 <100ms 绿 / <300ms 黄 / ≥300ms 红；亮 / 暗主题次级文字对比度提升至 WCAG AA。
- Quake 热键改为按键录制器（显示友好键名）；标签关闭确认由系统原生弹窗改为应用风格弹层；SSH 模态标题区分新建 / 编辑。

### 🧹 清理
- 删除前端从不调用的后端命令（`ssh_exec` / `pick_directory` / `save_group` / `get_groups` / `toggle_quake_window_cmd`）及相关数据结构。
- 移除约 30 个无引用的 npm 依赖及随早期 AI 功能删除遗留的 tailwind / postcss 配置。
- 删除设置中功能不存在的「数据库 / Docker 快捷键速查表」与「推介有奖」占位页。

### 🐛 修复
- 修复 Linux / macOS 构建失败（`SHELL_INTEGRATION_COUNTER` / `UNIX_EPOCH` 在 Unix 专属 shell 集成代码中实际仍被使用，此前被误删）。

## v0.4.0 - 2026-06-07

本版本参考开源项目 Tabby，合入整套终端 / 会话增强功能，并修复了服务器面板与若干稳定性问题。

### ✨ 终端与输入
- **终端内搜索**：可配置热键（默认 `Ctrl+Shift+H`）唤出搜索栏，支持查找下一个 / 上一个、匹配计数与高亮跳转（xterm SearchAddon）。
- **命令历史自动补全**：按 cwd / 主机 / 会话类型分域的历史排名建议 + 行内 ghost text，`Tab` / `→` 接受，`↑↓` / `Ctrl-N/P` 在多个候选间切换；基于 OSC 133 / OSC 7 捕获命令与工作目录。
- **代码片段库（Snippets）**：侧栏面板，增删改查并一键发送到终端，支持转义展开。
- **粘贴安全**：粘贴多行内容前弹窗确认（原生粘贴、右键菜单、上下文菜单统一走确认）。
- **输入广播**：状态栏开关，开启后将按键同时分发到所有已连接终端。
- **登录脚本**：SSH / 串口 / 本地会话连接后自动执行 `init_command`。
- **终端主题**：6 套可选配色方案 + auto，切换即时生效。

### 🧭 会话与导航
- **命令 / 连接面板**：全局热键（`Ctrl+Shift+F` / `Ctrl+Shift+B`）快速搜索并跳转会话。
- **快速连接**：轻量弹窗，解析 `user@host:port` 直接连接（不落库）。
- **会话 / 标签恢复**：启动时恢复上次打开的标签并自动重连（仅 ssh / 本地 / 串口）。
- **可配置全局快捷键**：设置内可编辑快捷键绑定（chord），持久化到设置。
- **分组级联默认值**：为分组设置可继承的默认连接字段，连接 / 重连时应用到会话。
- **2-pane 分屏（opt-in）**：并排显示两个终端；默认（不分屏）路径与原先逐字节一致。

### 🔌 隧道与转发
- **远程端口转发**（`tcpip_forward` + forwarded-tcpip 桥接）。
- **动态 SOCKS5 转发**（复用 direct-tcpip 桥接）。
- **SSH Agent 转发（-A）**：完整的 forwarded-agent 代理（opt-in）。

### 🪟 窗口与安全
- **Quake 下拉控制台**：全局热键唤出 / 收起的下拉式终端窗口（opt-in）。
- **主密码应用锁**：可选的 Argon2id 主密码锁，锁定时以全屏遮罩覆盖应用；不改动 keyring / secret 加密。

### 🐛 修复
- **服务器面板点击无反应**：`index.html` 的早期捕获阶段点击代理会拦截任意 `data-gw-action`，吞掉服务器面板按钮的 React `onClick` 并触发不存在的 `plugin:window|toggle_server_panel` IPC，使按钮完全无响应。现仅代理真正的窗口控制动作，应用级动作交还 React 处理。
- **打开服务器面板崩溃（Tokio panic）**：`start_server_metrics` 原为同步命令，在无 Tokio 运行时的主线程上 `tokio::spawn` 触发 panic（“there is no reactor running”），SSH 会话下打开服务器面板即整体崩溃；改为 async 命令在异步运行时执行。
- 远程转发绑定回环地址（`127.0.0.1`）而非所有接口；`vault_change_passphrase` 改为原子操作。

### 🎨 其他
- 全新原生 macOS 风格应用图标（圆角 squircle、透明四角、悬浮留白）。
- 新增针对真实 Linux `/proc` 数据的服务器监控解析回归测试。

## v0.3.0 - 2026-06-04

### ♻️ SSH 后端重写（russh）
- 将 SSH 后端从 libssh2（`ssh2` crate）整体重写为纯 Rust 异步实现 **russh 0.61 + russh-sftp**。旧实现用单线程切换阻塞模式并手动交错读写，频繁输入时会破坏发送流被服务器断开，导致终端静默卡死、无法输入（“频繁输入或回车必死”）。新架构下读、写在各自的异步任务中进行，不再切换阻塞模式、不再共享同一个 C 会话，从根本上消除该卡死。
- 交互式通道拆分为独立的读 / 写任务：被发送窗口阻塞的写不再饿死入站读取（保证 russh 能处理 `CHANNEL_WINDOW_ADJUST`），杜绝大量双向数据（大段粘贴 + 服务器刷屏）时的读写死锁。
- 单连接多路复用：SFTP、命令执行、服务器监控、端口转发改为同一连接上的独立通道，移除了仅为绕开 libssh2 并发限制而存在的辅助连接 / 监控连接 / SFTP worker 机制。
- 功能保持对等：交互式 Shell、SFTP、exec / 监控、本地端口转发、跳板机、SOCKS5 / HTTP 代理、Keepalive、主机指纹 / known_hosts 校验。

### 🔐 主机指纹校验
- 改为按密钥算法类型校验：同算法但指纹变化才视为「指纹变更」（可能的中间人）并告警；不同算法（例如旧版 libssh2 协商到 ECDSA、russh 协商到 Ed25519）按「新密钥」提示接受，与 OpenSSH 行为一致。修复了从旧版本升级后对已信任主机误报「主机指纹已变更」的问题。

### 🐛 稳定性修复
- keyboard-interactive 认证增加提示轮数上限：服务器无限重发提示时不再客户端死循环。
- `ssh_exec` / 结束远程进程增加超时兜底：命令不关闭通道时不再无限挂起。
- 会话关闭时取消仍在进行的端口转发桥接子任务，避免其持有共享连接句柄导致 socket 不被回收。
- 修复 Windows 构建：SSH agent 认证改为按平台连接（Unix 用 `$SSH_AUTH_SOCK`，Windows 用 OpenSSH 命名管道 / Pageant）；此前 `connect_env` 仅 Unix 可用，导致 Windows 平台编译失败。

## v0.2.2 - 2026-05-31

### 🐛 稳定性修复
- SSH 辅助连接（SFTP / 监控）增加读取超时：半开或卡死的连接不再让阻塞读永久占用辅助会话锁，避免该会话后续所有 SFTP / 执行命令死锁直到进程退出
- 服务器监控改用独立的 SSH 连接，不再与 SFTP 争用同一把锁；无法另建连接时（如一次性 TOTP）优雅回退到共享连接
- 监控首次静态探针增加超时，半开连接不再让服务器面板永久转圈
- 串口 `open` 改为幂等，关闭时 join 读取线程，消除重复打开导致的孤儿线程 / 端口句柄泄漏
- 修复指纹确认弹窗在标签页关闭时的 promise 闭包泄漏；分屏模式下也能正常弹出

### 🔐 凭据安全
- 解密失败时不再把 `enc:v1:` 密文当作密码返回（改为返回空并提示重新输入），避免无法诊断的登录失败
- 系统无密钥环、凭据将以明文存储时给出明确的界面提示，不再静默降级

### ⚡ 性能 / 响应
- 缩小终端容器对全局状态的订阅，减少无关状态变更引发的整树重渲染
- 串口读取轮询从 10ms 放宽到 150ms，空闲唤醒减少约一个数量级（接收延迟不变）
- 启动时在后台线程预热密钥环主密钥，避免冷启动阻塞主线程

### 🐛 功能修复
- 更新下载在二次检查返回空时不再永久卡在「下载中」
- 状态栏时钟改为定时刷新（此前会冻结）；移除占位的假天气
- 切换界面语言不再重启监控流 / 清空曲线历史
- 监控对缺少 `nproc` 的精简 Linux 主机更健壮：改用内核字符串判断是否受支持，不再误判为 unsupported

### 🎨 界面
- 终端光标 / 选区改为中性灰，替代原蓝色强调色
- 更新应用图标

### 🔧 CI / 构建
- 发布产物仅在打 tag（`v*`）时上传到 Release；PR / main 构建只做编译验证，消除并发构建上传同名产物到共享草稿 Release 的冲突

## v0.2.1 - 2026-05-30

### 🔧 CI / 构建修复
- 修复 macOS 发布构建失败（`tauri: command not found`，退出码 127）：`npm ci` 偶发产生不完整安装（npm/cli#4769「Exit handler never called!」会以退出码 0 结束却未链接 bin，并可能跳过可选的原生 CLI 二进制）。安装步骤改为重试，并以 `tauri --version` 真正可运行作为成功判定，而非仅检查 bin 文件存在

- Linux 构建显式安装 `libdbus-1-dev`（keyring / secret-service 的 dbus 链接），不再依赖 runner 镜像隐式提供

### 🧹 清理
- 移除未使用的 `lucide-ai` 依赖别名（指向 lucide-react 的重复拷贝，无任何源码引用）

> 本版本仅涉及 CI / 构建与依赖维护，应用功能与 v0.2.0 完全相同。

## v0.2.0 - 2026-05-30

### 🚀 重大修复：SSH 输入卡死（架构层面）
- 写入 / resize / 关闭命令移出 WebView 主线程（async + spawn_blocking），输入不再冻结界面

- 移除应用内 Auto Mode（自动确认）—— Claude Code 已原生提供该能力。删除其监听器、规则、开关、状态指示、日志面板、存储与全部设置项
